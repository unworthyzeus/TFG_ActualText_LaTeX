\chapter{Methodology / project development}

This chapter describes the dataset protocol, the evolution of the modelling
approach, and the final residual GMM-head model architecture. The emphasis is on decisions that
affect reproducibility: masking, splitting, conditioning, and the distinction
between raw deep prediction and prior-anchored residual prediction.

\section{Dataset protocol}

Each sample is a complete $513\times513$ pixel CKM. The model never receives a
random crop as an independent training city. Instead, the split is performed at
city level. For the distribution-first and final neural models the split uses
\texttt{split\_seed = 42}, with 15\% validation and 15\% test samples after
city assignment. This avoids the most common optimistic error in spatial radio
prediction: training on one part of a city and testing on another visually
similar part of the same city.

The ML-calibrated path-loss and spread priors do not require a validation set for early stopping or hyperparameter
selection, so they use a two-way split: 70\% train and 30\% test, with the
same \texttt{split\_seed = 42} and the same city-holdout assignment logic.
The training cities are therefore identical to those of the deep-learning
models; the 30\% prior-evaluation holdout corresponds to the union of the
15\% validation and 15\% test cities used by the neural models.

The receiver mask is:
\begin{equation}
  m_{\mathrm{ground}}(i,j) =
  \begin{cases}
    1, & \text{if } \mathrm{topology}(i,j)=0,\\
    0, & \text{otherwise.}
  \end{cases}
  \label{eq:ground_mask}
\end{equation}
All RMSE values in this thesis are pixel-weighted over valid ground pixels:
\begin{equation}
  \mathrm{RMSE}
  =
  \sqrt{
  \frac{\sum_{s,i,j} m_s(i,j)\left(\hat y_s(i,j)-y_s(i,j)\right)^2}
       {\sum_{s,i,j} m_s(i,j)}
  }.
  \label{eq:rmse}
\end{equation}
This is stricter than averaging per-sample RMSEs because large valid maps
contribute proportionally to the number of receiver pixels.

\section{From U-Net and cGAN to masked baselines}

The first implementation used a U-Net generator with a PatchGAN discriminator.
The generator predicted channel maps from topology-related inputs, while the
discriminator encouraged realistic local patches. This was useful as a
software prototype but not as a final scientific direction. The adversarial
term did not improve the physical errors that dominated validation RMSE, and it
made diagnosis harder.

The next phase focused on controlled engineering changes. Transposed
convolutions were replaced by bilinear upsampling and projection to reduce
checkerboard artifacts. Multi-scale supervision was added so that the model
could learn both global radial structure and local map details. The combined
decoder/multiscale run became
the strongest clean pre-prior path-loss baseline in this family.

After the masking correction, building pixels were excluded from both training loss and
validation metrics. This changed the interpretation of earlier numbers: any
method that implicitly benefited from easy building pixels was no longer
acceptable. The ground-only rule became an invariant for later models.

\section{Development lessons used by the methodology}

The work was not a single clean jump from U-Net to the final residual GMM-head model. The detailed numbering is kept in Appendix~A; the methodology only
retains the lessons that affect the final design. The final model is not only
the largest or latest network. It is the result of four negative lessons:
GANs were not reliable RMSE optimizers for this physical target; building
pixels had to be removed from the metric; pure CNNs underlearned radial LoS
structure; and NLoS errors were heavy-tailed enough to require regime-aware
or distribution-aware treatment. These lessons justify the final
prior-anchored formulation.

\section{Physical priors and PMHHNet}

The middle development phase moved from direct map prediction to residual learning, as
formulated in Eq.~\ref{eq:prior_residual}. This phase was heavily influenced by
the \texttt{PMNet} architecture \cite{pmnet2023}, which established the
viability of using U-Net-like backbones for path-loss map prediction in
terrestrial cellular networks.
\begin{equation}
  \hat{\textbf{Y}}
  =
  \textbf{P}_{\mathrm{phys}}
  +
  f_{\theta}\!\left(\mathcal{T}, h, \textbf{P}_{\mathrm{phys}}\right),
  \label{eq:prior_residual}
\end{equation}
where \(\textbf{P}_{\mathrm{phys}}\) is the analytical or calibrated prior map
and \(f_{\theta}\) predicts only the residual correction that the prior cannot
explain.

\subsection{Mathematical Evolution: From PMNet to PMHHNet}

The \texttt{PMHHNet} (Path-loss Map with High-frequency and Height-awareness
Network) was developed as a specific refinement of the \texttt{PMNet}
philosophy tailored for the UAV-enabled Air-to-Ground (A2G) scenario.

\paragraph{High-Frequency (HF) Stem.}
Standard convolutional encoders tend to smooth out sharp building transitions
during downsampling. To counter this, \texttt{PMHHNet} introduces a dedicated
high-frequency stem. Let $\mathcal{T}$ be the topology map. We compute the
discrete Laplacian $|\nabla^2 \mathcal{T}|$ to identify building edges and
shadow boundaries. The initial feature map $\textbf{X}_0$ is computed as:
\begin{equation}
  \textbf{X}_0 = \mathrm{Stem}(\mathcal{T}) + \mathrm{Proj}(|\nabla^2 \mathcal{T}|)
  \label{eq:hf_stem}
\end{equation}
where $\mathrm{Proj}$ is a $1\times1$ convolution that maps the edge-detector
output to the stem's feature depth. This ensures that building geometry is
strongly injected into the earliest layers of the encoder.

\paragraph{Continuous Sinusoidal FiLM Conditioning.}
While \texttt{PMNet} was designed for fixed base-station heights, UAVs require
continuous height generalization. \texttt{PMHHNet} utilizes Feature-wise Linear
Modulation (FiLM) \cite{perez2018film} to inject the scalar height $h$.
The height is first mapped to a sinusoidal embedding $\textbf{e}(h) \in \mathbb{R}^K$
(Eq.~\ref{eq:sin_film}). For any feature map $\textbf{X} \in \mathbb{R}^{C \times H \times W}$
at depth $l$, the conditioned map $\textbf{X}'$ is:
\begin{equation}
  \textbf{e}(h)
  =
  \left[
    \sin(\omega_1 \tilde{h}), \cos(\omega_1 \tilde{h}), \ldots,
    \sin(\omega_{K/2} \tilde{h}), \cos(\omega_{K/2} \tilde{h})
  \right],
  \qquad
  \tilde{h} = \frac{h-h_{\min}}{h_{\max}-h_{\min}},
  \label{eq:sin_film}
\end{equation}
\begin{equation}
  \textbf{X}' = \textbf{X} \odot (1 + \textbf{g}_l(\textbf{e}(h))) + \textbf{b}_l(\textbf{e}(h))
  \label{eq:film_math}
\end{equation}
where $\textbf{g}_l$ and $\textbf{b}_l$ are learned linear projections mapping the
embedding to the channel dimension $C$. This pattern, inspired by time-step
conditioning in diffusion models \cite{dhariwal2021diffusion}, allows the
network to modulate its internal representation dynamically based on the UAV's
altitude.

\subsection{Limitations and Legacy}

Despite these improvements, the early residual phase plateaued at about
\SI{9.4}{dB}. The primary bottleneck was the "soft" nature of the
residual learning; the network often under-compensated for the physical
propagation law in NLoS regions.

\begin{quote}
\textit{Legacy of PMHHNet:} While the full architecture was eventually superseded
by the final prior-anchored GMM approach, its core components---the
HF stem and sinusoidal FiLM---remain highly effective for point-prediction
tasks or real-time implementations where the distributional complexity of a GMM
is not required.
\end{quote}

\section{Distribution-first modelling}
\label{sec:dist_first}

The decisive late-stage diagnosis was that NLoS path loss was not merely a
harder regression target; it possessed a fundamentally different error
distribution.  This diagnosis came from comparing predicted and ground-truth
histograms: the maps showed where the model was wrong, but the histograms
showed \emph{what} the model had misunderstood.  The central idea of the late
pipeline was therefore to identify the target distribution before asking a
network to perform pixel-by-pixel prediction. The distribution-first path-loss
model addressed this by
predicting the parameters of a Gaussian Mixture Model (GMM) rather than a
simple point estimate.

For path loss, the distribution-first model utilized 12 specialized experts: six topology classes partitioned by LoS
and NLoS regions. As shown in Figure~\ref{fig:try76_system}, Stage A acts as a distribution head, predicting the global GMM parameters
$(\pi_k,\mu_k,\sigma_k)$ from map features and altitude conditioning.
Stage B then decodes these priors spatially, predicting per-pixel soft component assignments and within-component
offsets. The final map reconstruction can be written as:
\begin{equation}
  \hat y(i,j)
  =
  \sum_{k=1}^{K} p_k(i,j)
  \left(\mu_k + z_k(i,j)\sigma_k\right).
  \label{eq:gmm_reconstruction}
\end{equation}

The spread counterpart extended this distribution-first philosophy to delay spread and angular spread. As illustrated in Figure~\ref{fig:try77_system}, these targets
are not cleanly Gaussian; their empirical distributions resemble a near-zero spike (representing the LoS direct path) combined with a
heavy tail (representing NLoS scattering). Standard Mean Squared Error (MSE) objectives tend to underrepresent these rare
but physically significant heavy-tail spread values, necessitating a dedicated mixture head.

\subsection{How to construct a good deep-learning model for radio maps}
\label{sec:good_dl_model_design}

The main lesson from the deep-learning experiments is that the architecture is
secondary to the question: \emph{what statistical object is the network being
asked to predict?} Path loss, delay spread, and angular spread are all dense
radio maps, but they are not the same kind of target. Treating them as three
ordinary image-to-image regression problems led to the repeated failures of
the early U-Net, cGAN, PMNet, and PMHHNet branches. The successful late
designs first identified the distribution of each target and only then chose
the model, loss, routing, and conditioning.  The literature review on MDN and
GMM channel models was then carried out while writing the thesis, after this
distribution-first idea had already emerged from the CKM diagnostics.  That
review showed that mixture-density ideas exist in wireless channel modelling,
but that their use as a global distribution guide for dense radio-map
prediction remains an innovative contribution of this thesis.

\subsubsection{A prior is useful, but not the core requirement}

The project initially made the physical prior look indispensable because the
best mid-project models were residual-over-prior systems. However, the
distribution-first path-loss model showed that the prior was not the essential
ingredient. It
used no COST-231 prior, no enhanced two-ray prior, no knife-edge channel, and
no residual-over-prior decomposition. It still obtained NLoS path-loss results
close to the final prior family because it modelled the NLoS distribution
directly. This changes the interpretation of the final path-loss prior,
spread prior, and residual GMM-head designs. The priors are not magic
external knowledge. They are convenient, stable ways of estimating the same
large-scale statistics that a distribution-aware neural model can also learn
if it is given the right objective and enough evidence.

This is also why the final pipeline reintroduces priors without contradicting
the distribution-first lesson. The order in which the evidence arrived matters:
the distribution-first model came immediately after the histogram
diagnosis, which showed that the dominant failure mode was an
output-distribution mismatch rather than a missing physical channel. The
first response was to fix the deep-learning side and prove that NLoS path
loss can be modelled without extra physics. Once that point was made, the
question became which part of the pipeline is cheapest and most auditable.
For LoS path loss the answer turned out to be a short analytic model with a
handful of fitted constants; for delay and angular spread
the answer was a log-domain ridge regression over geometry and morphology
features; the final residual GMM-head model then anchors a shared neural
residual head to both. The priors in the final pipeline are therefore an
efficiency and stability device, not a substitute for distribution-aware
modelling.

Therefore, a prior should be seen as an efficiency and stability device, not
as a substitute for modelling the target correctly. A good prior reduces the
burden on the network by supplying a strong mean prediction, especially for
LoS path loss and log-spread baselines. A good GMM or mixture-density network
can instead learn that mean and uncertainty internally. Both strategies work
when the model knows the support, modes, masks, and regime split of the target.
Both fail when the training objective compresses distinct physical regimes
into one averaged map.

\subsubsection{Path loss: LoS has structure, NLoS has a narrow but noisy law with outliers}

For path loss, the LoS and NLoS targets have different meanings. LoS is not
just the easy subset. It contains a strong deterministic spatial structure:
distance decay, UAV-height dependence, and in CKM a visible coherent two-ray
ring pattern. This is why even imperfect models often reached LoS RMSE around
\SI{4}{dB}. A CNN can learn a smooth radial law from the distance map and
topology context, but it cannot invent coherent interference if the loss and
features do not encourage that structure. The final prior solved this with an
explicit enhanced two-ray model; a neural model could also solve it, but it
would need enough capacity and a loss that rewards the ring structure rather
than only the pixel average.

NLoS is different. In the valid ground pixels of CKM, the NLoS path-loss
marginal is not wildly spread over the full \SIrange{0}{180}{dB} output
interval. It is relatively narrow, concentrated around the blocked-link
attenuation level, but it is spatially noisy, contains outliers, and is hard
to place. A pixel just behind a corner, a pixel inside a street canyon, and a
pixel in a shallow shadow can have similar global statistics but different
local errors. The model therefore faces two tasks at once: estimating the
right NLoS value range and locating the rare high-error blocked pockets. This
is why ordinary point regression failed in a confusing way. The model did not
need to predict an arbitrarily broad distribution, but it did need to know
\emph{which} NLoS distribution it was in, where each pixel sat inside it, and
which pixels belonged to the tail. The old PMHHNet family instead produced
catastrophic NLoS mode collapse: predicted NLoS means around \SI{36.5}{dB}
against target means around \SI{107.1}{dB}. The issue was not that NLoS was
impossible; it was that the network and loss were representing it as an
ordinary residual image.

This explains the distribution-first design. Stage A estimates the per-sample
target distribution through a GMM, while Stage B predicts the spatial
assignment of pixels to mixture components. The first part answers
``what values should this map contain?'' The second answers ``where should
those values go?'' This separation is exactly what direct MSE, Huber, and
plain RMSE heads were missing.  In practical terms, the GMM head gives the
model a global description of the distribution it is about to draw; the decoder
then performs the local placement.  This is why distribution prediction can
help a radio-map model even when the final deliverable is still a deterministic
per-pixel map.

\subsubsection{Delay spread and angular spread: NLoS is the important regime}

Delay spread and angular spread are even more distributional than path loss.
In LoS regions, the direct path dominates: delay spread can be very small, and
angular spread can collapse toward a narrow direction. In NLoS regions, the
same quantities become much more meaningful because the receiver sees
diffracted, reflected, and scattered components. A high delay spread or a wide
angular spread is not merely an error-prone outlier; it is the physical signal
that multipath has become important.

The histogram study makes this visible. Figure~\ref{fig:target_prediction_histograms}
aggregates the validation histograms used in the distribution-first diagnosis. Path-loss
NLoS shows the old mode-collapse failure: the target NLoS mass sits around
\SI{107}{dB}, while the old prediction histogram puts too much mass at much
smaller values. The spread targets are different. Their targets contain a
large near-zero or low-value component plus a long, irregular tail. The
prediction histograms learn the bulk of the distribution but truncate much of
that tail. In the histogram table behind the figure, delay-spread target
\(p_{99}\) is about \SI{142}{ns}, while the corresponding prediction
\(p_{99}\) is only about \SI{47}{ns}; angular-spread target \(p_{99}\) is
about \(74^\circ\), while the prediction \(p_{99}\) is about \(43^\circ\).

This is why the LoS/NLoS interpretation of the spread metrics is not the same
as for path loss. For path loss, LoS is the clean deterministic branch and
NLoS is the hard blocked branch. For delay and angular spread, NLoS is often
where the target contains the most useful channel information. It is therefore
physically consistent that the final spread models can have lower or
competitive NLoS RMSE: the NLoS spread distribution contains stronger
geometry-driven signals, while the LoS distribution is dominated by a
near-zero spike plus quantisation and simulator noise. A model that only
optimizes global RMSE may overfit the spike and underrepresent the physically
important tail. The correct approach is to model the spike and the tail
separately, as in the distribution-first spike-plus-tail design and the
final log-domain regime-wise calibration.

The radio-map images show the same phenomenon spatially. For spread targets,
the model often captures the low-frequency image structure: broad regions of
small spread, broad NLoS areas, and the general city-scale morphology. What it
misses are local high-spread islands, such as a zone where rays may bounce off
a tall or especially reflective building and arrive through a small number of
strong delayed paths. These are not random visual speckles, but they are also
not fully predictable from coarse morphology. A model must choose its tail
policy. It can be conservative in the statistical sense and predict few
outliers, which gives many false negatives in the tail; or it can be
conservative in the engineering sense and over-predict possible high-spread
zones, accepting more false positives. The results in this thesis mostly use
the first policy: avoid hallucinating too many high-spread islands. For future
network-planning use, the second policy may be preferable, because engineering
design often cares more about not missing worst-case multipath than about
avoiding every false alarm.

\subsubsection{Why the cGAN line failed}

The cGAN experiments failed because adversarial image realism was the wrong
pressure for this problem. A radio map is not judged by whether it looks like
a plausible texture; it is judged by physically calibrated errors under a
ground mask, separated by LoS/NLoS, topology, and antenna height. The
discriminator encouraged sharper outputs and plausible local detail, but it
did not know that a \SI{30}{dB} NLoS underprediction is much worse than a
visually smooth map. GAN losses are also weak at preserving calibrated
absolute units: they can improve perceptual sharpness while leaving the mean,
tail, or regime-specific error wrong.

This mattered especially for NLoS and spread targets. The rare high-error
regions are not generic high-frequency texture. They are tied to blocked
streets, corners, local density, elevation angle, and scattering. A cGAN can
hallucinate texture, but it cannot recover a missing statistical tail unless
the objective explicitly demands the correct tail mass and the correct
physical units. In this project, once the evaluation was made strict
city-holdout and regime-aware, the cGAN family did not solve the real
failure mode.

\subsubsection{Why PMNet and PMHHNet were not enough}

PMNet and PMHHNet were much better ideas than the early cGAN line because they
matched radio-map structure more closely. PMNet-style context aggregation
helps with long-range spatial dependence. PMHHNet added high-frequency
topology information, height FiLM conditioning, GroupNorm, topology experts,
and explicit physical channels. These changes explain why LoS became good and
overall RMSE improved substantially.

However, our implementations still failed for two reasons. First, several
branches had training-design defects: the main full-resolution reconstruction
loss was disabled or weak in some configurations, Huber losses masked large NLoS tail
errors, over-regularization made predictions collapse toward a baseline,
corridor weighting over-emphasized the centre of the map, and one PMHHNet path
temporarily failed to inject the high-frequency topology signal into the early
stem. These were implementation problems, and fixing them improved stability.

Second, even after those fixes, PMNet and PMHHNet were still mostly
point-regression architectures. They predicted one value per pixel without
first estimating the sample-level target distribution. That is sufficient for
smooth LoS path loss, but not for NLoS path loss or spread tails. The network
could reduce average error by predicting a safe central value, while RMSE
remained dominated by the pixels whose physical regime had been averaged
away. This is the reason the project moved from ``better backbone'' to
``better probabilistic target model''.

\subsubsection{Design rules that survived}

The final practical rules for deep-learning radio-map prediction are the
following.

\begin{enumerate}
\item \textbf{Identify the target distribution first.} Decide whether the
      target is smooth, narrow, multi-modal, spike-plus-tail, or
      heavy-tailed before choosing MSE, Huber, GMM, NLL, or residual losses.
\item \textbf{Separate LoS and NLoS semantically, not only as metrics.}
      LoS path loss is mostly deterministic structure; NLoS path loss is a
      blocked-link distribution-placement problem; NLoS spreads are often
      the physically informative part of the target.
\item \textbf{Report regime metrics throughout training.} A good global RMSE
      can hide a broken NLoS branch. LoS, NLoS, topology class, antenna
      height, and target-specific metrics must be visible in validation.
\item \textbf{Use priors when they reduce burden, not because they are
      mandatory.} A frozen prior is useful when it gives a reliable baseline,
      but a distribution-aware model can also work without it if it learns the
      correct marginal and spatial assignment.
\item \textbf{Do not use visual realism as a proxy for radio accuracy.}
      Adversarial or perceptual losses must be subordinate to calibrated
      physical-unit losses and distribution checks.
\item \textbf{Keep masks, units, and support explicit.} Ground masking,
      no-data handling, clipping ranges, log transforms for spreads, and
      city-holdout splits are part of the model definition, not preprocessing
      details.
\item \textbf{Choose an explicit tail-risk policy.} For delay and angular
      spread, false-negative outliers make maps look cleaner but can miss
      worst-case multipath; false-positive outliers are less pretty but may be
      safer for conservative engineering planning.
\end{enumerate}

\section{GMM component count and distribution choice}
\label{sec:gmm_k_choice}

The Stage-A distribution heads in the distribution-first models
represent each per-sample target marginal as a parametric family. Choosing
the family and its number of components $K$ is not a free hyperparameter; it
is constrained by the empirical shape of the CKM histograms. A dedicated
distribution-ranking diagnostic fits a fixed shortlist of parametric models
to the aggregated validation histograms of every target under each topology
class and LoS/NLoS split, and ranks them by the KL divergence
\(\mathrm{KL}(p_{\mathrm{empirical}}\;||\;p_{\mathrm{model}})\).  The shortlist
contains Gaussian, Laplace, skew-Normal, log-Normal, Gamma, Weibull, a
spike-plus-exponential model for near-degenerate targets, and Gaussian
mixtures with $K\in\{2,3,4,5\}$.

Three consequences follow. First, a single Gaussian (or any unimodal head) is
never the best family for NLoS path loss or for the two spreads. This is
exactly the regime in which the earlier PMHHNet family produced mode collapse
with a single-output MSE head. Second, the number of components that wins is
consistently $K\in\{4, 5\}$ for the raw path-loss and spread targets, not
$K=3$. The Stage-A heads in this thesis therefore use $K=5$ by default on
the target-distribution experts, even though the code supports
$K\in\{2,\dots,5\}$. The final \texttt{num\_components} field
configuration (\(K=3\)) controls a different quantity: the shared backbone's
residual distribution head after the frozen priors have already
explained the main LoS, NLoS, delay-spread, and angular-spread structure. In
that residual space, three modes are sufficient to model negative correction,
near-zero correction, and positive correction, while avoiding a larger mixture
that would be harder to stabilize during the final multi-task training. Third, the
angular-spread target is better served by a native spike-plus-exponential
head than by a GMM, because its marginal is effectively degenerate at
\ang{0}; this is implemented directly in the distribution-first angular-spread
experts.

\section{Final residual GMM-head model training progression}
\label{sec:try80_training_progression}

The final residual GMM-head model was not obtained in a single run. Training began with
the joint prior-anchored residual design, but the first complete runs produced
only small improvements over the frozen priors. As a diagnostic,
this showed that the priors were already strong, but also that the residual
model was initially too constrained to correct the remaining errors, especially
for path loss.

The training path then followed four stages:

\begin{enumerate}
\item \textbf{Big.} The first full version used a shared multi-task
U-Net with \(96\) base channels and a \(128\)-dimensional FiLM condition
vector. Its residual caps were conservative, especially for path loss
(\(\pm2\) dB in LoS and \(\pm4\) dB in NLoS), and the objective retained the
complete probabilistic bundle: GMM likelihood, distribution matching,
prior anchoring, prior guard, RMSE, and MAE. This stage mainly tested whether
a residual network could improve the frozen priors without degrading them.

\item \textbf{Big with relaxed residuals.} The second stage kept the same
network capacity but loosened the allowed correction range. The path-loss caps
increased to \(\pm5\) dB in LoS and \(\pm20\) dB in NLoS, with wider spread
caps as well. This made the residual head less constrained and produced a
clearer, but still limited, improvement.

\item \textbf{Huge with relaxed residuals.} The third stage increased model
capacity to \(128\) base channels and a \(160\)-dimensional conditioning
vector, while keeping the relaxed residual philosophy. It also placed more
weight on the deterministic RMSE term. This stage tested whether the remaining
gap was capacity-limited rather than only cap-limited. The larger model helped
the spread tasks, but path-loss improvement remained too small while the full
probabilistic objective was still active.

\item \textbf{Huge path-loss fine-tuning.} The final stage kept the huge
architecture and relaxed caps, lowered the learning rate, and changed the
objective into a short RMSE-dominant fine-tune. The probabilistic terms were
disabled, the prior guard and anchor were kept only as light stabilizers, and
path loss received the strongest task weight. This is the checkpoint used for
the reported final results: the GMM structure remains the residual output
head, but model selection is driven by deterministic validation performance.
\end{enumerate}

For this reason, the methodology does not treat training curves themselves as
evidence for the final claim. The relevant evidence is the same city-holdout
validation and test evaluation used throughout the thesis, reported in
Chapter~\ref{chap:results}. Training curves were used only as diagnostics for
deciding which stage should be continued or replaced.
